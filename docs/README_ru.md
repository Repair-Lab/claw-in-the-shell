## 🌊 Что такое GhostShell?

**GhostShell — это реляционная операционная система на основе ИИ.** В то время как проекты вроде OpenClaw работают *на* системе, GhostShell **является** системой. Он превращает базу данных PostgreSQL в живой организм, где аппаратные драйверы, файловые системы и модели ИИ («призраки») общаются через SQL-таблицы.

Каждая мысль. Каждое перемещение файла. Каждый аппаратный импульс. Всё это — ACID-совместимые транзакции базы данных. Несокрушимо. Безопасно. Согласованно.

## 🔥 Почему GhostShell, а не OpenClaw?

| | OpenClaw | GhostShell OS |
|---|---|---|
| **Архитектура** | Приложение на системе | **Является** системой |
| **Сохранность данных** | Энергозависимая память | ACID-транзакции — каждая мысль постоянна |
| **Оборудование** | Внешние API | Оборудование-как-Таблица — `UPDATE cpu SET governor='performance'` |
| **Модели ИИ** | Одна модель, требуется перезагрузка | Горячая замена призраков — смена LLM без потери контекста |
| **Безопасность** | Уровень приложения | 3 уровня неизменяемости: Ядро → Среда выполнения → Призрак |
| **Самовосстановление** | Ручное | Автономный конвейер восстановления с одобрением человека |

---

## 🔒 Три уровня безопасности

**Призрак может чинить — но никогда не перестраивать.** Каждое предложенное изменение проходит через:

```
Призрак предлагает → Человек одобряет → SECURITY DEFINER выполняет → Журнал аудита
```
