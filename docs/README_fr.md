## 🌊 Qu'est-ce que GhostShell ?

**GhostShell est un système d'exploitation IA relationnel.** Alors que des projets comme OpenClaw fonctionnent *sur* un système, GhostShell **est** le système. Il transforme une base de données PostgreSQL en un organisme vivant où pilotes matériels, systèmes de fichiers et modèles d'IA (« Ghosts ») communiquent via des tables SQL.

Chaque pensée. Chaque déplacement de fichier. Chaque impulsion matérielle. Tout cela — des transactions de base de données conformes ACID. Indestructible. Sécurisé. Cohérent.

---

## 🔥 Pourquoi GhostShell et non OpenClaw ?

| | OpenClaw | GhostShell OS |
|---|---|---|
| **Architecture** | Application sur un système | **Est** le système |
| **Persistance** | Mémoire volatile | Transactions ACID — chaque pensée est permanente |
| **Matériel** | APIs externes | Matériel-en-Table — `UPDATE cpu SET governor='performance'` |
| **Modèles IA** | Modèle unique, redémarrage requis | Hot-Swap Ghosts — changez de LLM sans perdre le contexte |
| **Sécurité** | Niveau application | 3 couches d'immuabilité : Noyau → Runtime → Ghost |
| **Auto-réparation** | Manuelle | Pipeline de réparation autonome avec approbation humaine |

---

## 🔒 Trois Couches de Sécurité

**Le Ghost peut réparer — mais jamais reconstruire.** Chaque changement proposé passe par :

```
Ghost propose → Humain approuve → SECURITY DEFINER exécute → Journal d'audit
```
