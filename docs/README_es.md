## 🌊 ¿Qué es GhostShell?

**GhostShell es un sistema operativo de IA relacional.** Mientras que proyectos como OpenClaw se ejecutan *sobre* un sistema, GhostShell **es** el sistema. Transforma una base de datos PostgreSQL en un organismo vivo donde controladores de hardware, sistemas de archivos y modelos de IA ("Ghosts") se comunican a través de tablas SQL.

Cada pensamiento. Cada movimiento de archivo. Cada impulso de hardware. Todo — transacciones de base de datos compatibles con ACID. Indestructible. Seguro. Consistente.

---

## 🔥 ¿Por qué GhostShell y no OpenClaw?

| | OpenClaw | GhostShell OS |
|---|---|---|
| **Arquitectura** | Aplicación sobre un sistema | **Es** el sistema |
| **Persistencia** | Memoria volátil | Transacciones ACID — cada pensamiento es permanente |
| **Hardware** | APIs externas | Hardware-como-Tabla — `UPDATE cpu SET governor='performance'` |
| **Modelos IA** | Modelo único, requiere reinicio | Hot-Swap Ghosts — cambia LLMs sin perder contexto |
| **Seguridad** | Nivel de aplicación | 3 capas de inmutabilidad: Núcleo → Runtime → Ghost |
| **Autorreparación** | Manual | Pipeline de reparación autónomo con aprobación humana |

---

## 🔒 Tres Capas de Seguridad

**El Ghost puede reparar — pero nunca reconstruir.** Cada cambio propuesto pasa por:

```
Ghost propone → Humano aprueba → SECURITY DEFINER ejecuta → Registro de auditoría
```

## 🦾 Características

- [x] **Hardware-como-Tabla** — Controla ventiladores, frecuencia CPU y discos por `SQL UPDATE`
- [x] **17 Aplicaciones de Escritorio** — Ghost Chat, Tienda de Software, Gestor LLM y más
- [x] **Hot-Swap Ghosts** — Cambia tu LLM en tiempo de ejecución sin perder contexto
- [x] **Protección de Inmutabilidad** — 176 huellas de esquema, registro de violaciones
- [x] **Pipeline de Reparación** — Ghost propone → Humano aprueba → Ejecución segura
- [x] **Puente OpenClaw** — Importa tus habilidades OpenClaw a un entorno más seguro
- [x] **Métricas en Tiempo Real** — CPU, RAM, GPU, temperatura por WebSocket
- [x] **Seguridad a Nivel de Fila** — 71 tablas con políticas RLS en 5 roles
- [ ] **Codificación Autónoma** *(En Progreso)* — Ghost escribe sus propias migraciones SQL
