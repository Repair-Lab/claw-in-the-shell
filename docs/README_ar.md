## 🌊 ما هو GhostShell؟

**GhostShell هو نظام تشغيل ذكاء اصطناعي علائقي.** بينما تعمل مشاريع مثل OpenClaw *فوق* نظام ما، فإن GhostShell **هو** النظام نفسه. يحوّل قاعدة بيانات PostgreSQL إلى كائن حي حيث تتواصل برامج تشغيل الأجهزة وأنظمة الملفات ونماذج الذكاء الاصطناعي ("الأشباح") عبر جداول SQL.

كل فكرة. كل نقل ملف. كل نبضة عتاد. كلها — معاملات قواعد بيانات متوافقة مع ACID. غير قابلة للتدمير. آمنة. متسقة.

---

## 🔥 لماذا GhostShell وليس OpenClaw؟

| | OpenClaw | GhostShell OS |
|---|---|---|
| **البنية** | تطبيق على نظام | **هو** النظام |
| **استمرارية البيانات** | ذاكرة متطايرة | معاملات ACID — كل فكرة دائمة |
| **العتاد** | واجهات برمجة خارجية | العتاد-كجدول — `UPDATE cpu SET governor='performance'` |
| **نماذج الذكاء** | نموذج واحد، يتطلب إعادة تشغيل | تبديل ساخن للأشباح — غيّر LLM دون فقدان السياق |
| **الأمان** | مستوى التطبيق | ٣ طبقات ثبات: النواة ← وقت التشغيل ← الشبح |
| **الإصلاح الذاتي** | يدوي | خط أنابيب إصلاح ذاتي مع موافقة بشرية |

## 🔒 ثلاث طبقات أمان

**الشبح يمكنه الإصلاح — لكن لا يمكنه إعادة البناء أبداً.** كل تغيير مقترح يمر عبر:

```
الشبح يقترح ← الإنسان يوافق ← SECURITY DEFINER ينفذ ← سجل التدقيق
```

## 🦾 المميزات

- [x] **العتاد-كجدول** — تحكم بالمراوح وتردد المعالج والأقراص عبر `SQL UPDATE`
- [x] **١٧ تطبيق سطح مكتب** — محادثة الشبح، متجر البرمجيات، مدير LLM والمزيد
- [x] **تبديل ساخن للأشباح** — غيّر LLM أثناء التشغيل دون فقدان السياق
- [x] **حماية الثبات** — ١٧٦ بصمة مخطط، تسجيل الانتهاكات
- [x] **خط أنابيب الإصلاح** — الشبح يقترح ← الإنسان يوافق ← تنفيذ آمن
- [x] **جسر OpenClaw** — استيراد مهارات OpenClaw إلى بيئة أكثر أماناً
- [x] **مقاييس فورية** — المعالج، الذاكرة، GPU، الحرارة عبر WebSocket
- [x] **أمان على مستوى الصف** — ٧١ جدول بسياسات RLS عبر ٥ أدوار
- [ ] **البرمجة الذاتية** *(جارٍ)* — الشبح يكتب ترحيلات SQL الخاصة به
- [ ] **تكامل الرؤية** *(مخطط)* — تحليل فيديو فوري في `media_metadata`
- [ ] **أشباح موزعة** *(مخطط)* — عدة نسخ شبح عبر العقد
